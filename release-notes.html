Bug Fixes
  • Account Templates - new Identities could not be added in the new version of the settings dialog. The API used to write legacy settings to the global configuration database wasn't able to create any entries without default, which affects any Identities created by the user, including the default identity of a new mail account. This means such settings (template and quotes) could not be modified and saved unless the identity existed before the conversion of SmartTemplates settings to be web extension compatible. [issue 300] This problem is now patched and back to work as expected.
